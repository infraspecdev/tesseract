E1 Input Validation & Pydantic Models
E1-S1 Define strict Pydantic request/response models high 2 pts

Create TaskCreate, TaskUpdate, and TaskResponse models with field validation (max lengths, status enum, required fields).

5 tasks
  • Add TaskStatus enum (open, in_progress, done) to models.py
  • Create TaskCreate model with title (str, max_length=200, required), description (str, max_length=2000, optional), status (TaskStatus, default=open)
  • Create TaskUpdate model with all fields optional
  • Create TaskResponse model extending TaskCreate with id field
  • Add unit tests for model validation

Acceptance Criteria

  • TaskCreate rejects empty title
  • TaskCreate rejects title longer than 200 characters
  • TaskCreate rejects invalid status values
  • TaskUpdate allows partial updates (all fields optional)
  • TaskResponse includes id field
  • All models have type-safe status enum
E1-S2 Wire Pydantic models into route handlers high 2 pts depends: E1-S1

Replace dict parameters in task routes with Pydantic models. Return typed responses.

5 tasks
  • Update create_task to accept TaskCreate, return TaskResponse
  • Update update_task to accept TaskUpdate, return TaskResponse
  • Update list_tasks to return list[TaskResponse]
  • Update get_task to return TaskResponse
  • Verify FastAPI auto-generates OpenAPI schema with model types

Acceptance Criteria

  • POST /tasks/ with invalid body returns 422 with field-level errors
  • PUT /tasks/{id} with invalid status returns 422
  • GET /tasks/ returns typed response array
  • OpenAPI docs at /docs reflect new models

E3 JWT Authentication & Authorization
E3-S1 Implement JWT token creation and verification high 3 pts

Create auth module with JWT encode/decode utilities using PyJWT. Support configurable secret and expiration.

6 tasks
  • Add PyJWT to dependencies
  • Create src/auth.py with create_access_token(subject, role, expires_delta)
  • Add verify_token(token) function returning decoded payload or raising HTTPException(401)
  • Load JWT_SECRET from environment variable (fail fast if missing)
  • Add JWT_ALGORITHM and ACCESS_TOKEN_EXPIRE_MINUTES settings
  • Add unit tests for token creation, verification, and expiration

Acceptance Criteria

  • create_access_token returns valid JWT with sub, role, and exp claims
  • verify_token decodes valid tokens and returns payload
  • verify_token raises 401 for expired tokens
  • verify_token raises 401 for tokens with invalid signature
  • JWT_SECRET is required (not hardcoded)
E3-S2 Create FastAPI auth dependencies high 2 pts depends: E3-S1

Build reusable Depends() callables for extracting and validating JWT from Authorization header.

5 tasks
  • Create get_current_user dependency using OAuth2PasswordBearer
  • Extract Bearer token from Authorization header
  • Decode token and return user payload
  • Create require_role(role) dependency factory
  • Return 401 for missing/invalid tokens, 403 for insufficient role

Acceptance Criteria

  • get_current_user extracts user from valid Bearer token
  • Missing Authorization header returns 401
  • Invalid token returns 401 with {"detail": "Invalid credentials"}
  • require_role('admin') returns 403 for non-admin users
  • OpenAPI docs show lock icon on protected endpoints
E3-S3 Protect task mutation endpoints high 2 pts depends: E3-S2, E1-S2

Apply auth dependencies to POST, PUT, DELETE endpoints. Keep GET endpoints public.

5 tasks
  • Add get_current_user dependency to create_task
  • Add get_current_user dependency to update_task
  • Add require_role('admin') to delete_task
  • Keep list_tasks and get_task public
  • Add integration tests with valid/invalid/missing tokens

Acceptance Criteria

  • POST /tasks/ without token returns 401
  • POST /tasks/ with valid token returns 201
  • DELETE /tasks/{id} without token returns 401
  • GET /tasks/ works without authentication
  • GET /tasks/{id} works without authentication
E3-S4 Add login endpoint for token issuance medium 2 pts depends: E3-S1

Create a simple /auth/token endpoint that issues JWTs using an in-memory user store (demo only).

6 tasks
  • Create src/routes/auth.py with POST /auth/token endpoint
  • Accept username/password via OAuth2PasswordRequestForm
  • Validate against simple in-memory user store
  • Return {"access_token": "...", "token_type": "bearer"}
  • Register auth router in main.py
  • Add tests for login success and failure

Acceptance Criteria

  • POST /auth/token with valid credentials returns access_token
  • POST /auth/token with invalid credentials returns 401
  • Returned token is valid and decodable
  • Token includes sub and role claims
