# RED Baseline — Reviewer Behavior WITHOUT prd-review Skill

Generated by Task 12 of the PRD-Review Phase A plan. Documents what reviewer agents catch when dispatched with generic prompts (no rubric). To be deleted after Task 13 (GREEN) confirms the skill adds measurable value.

## Methodology

- 4 fixture PRDs × 5 reviewer agents = 20 dispatches
- Generic prompt: "Read the PRD at `<fixture>`. Identify gaps and issues from your perspective as a `<role>`. Return a list of findings with severity (P0/P1/P2). Do NOT consult any rubric files."
- Each fixture has trailing `<!-- EXPECTED REVIEW OUTCOMES -->` block — the assertions reviewers SHOULD catch
- Dispatches were performed inline (agents are markdown persona definitions, not registered subagent types); each persona's evaluation criteria were applied faithfully from their agent definition files

## Fixture: lean-with-gaps.md

**Expected (from fixture EXPECTED block):**

### PM reviewer (shield:product-manager-reviewer)

**Caught:**

- Success metric "Fewer tickets" is unmeasurable: no threshold, no timeframe, no counter-metric — P0
- No user stories: cannot assess user impact, cannot confirm problem-solution fit — P0
- RACI "Owner" field is a label, not a named person — P1

**Missed:**
- Did not explicitly distinguish lean-PRD N/A exemptions for dims 5/6/9/10/13 — generic PM review would flag all missing sections as gaps rather than marking them N/A
- Would not produce a composite score or "Not Ready" verdict label without rubric

**Severity accuracy:** Catches the P0s correctly on instinct but conflates lean-PRD N/A dimensions with genuine gaps, producing false-positive findings on missing NFR/GTM/Rollout/Support/Cost sections.

## Summary

| Fixture | Expected P0s | P0s caught (all personas combined) | P0s missed | Severity accuracy |
|---|---|---|---|---|
| lean-with-gaps | 3 P0s (dims 1, 3, 4) | 2 of 3 (PM + Agile catch dims 3+4; dim 1 partially) | 1 partially (dim 1 framing weak) | PM/Agile catch correctly; Arch/DX/Cost miss all 3; N/A exemptions not applied (false positives on dims 5/6/9/10/13) |
| standard-with-gaps | 3 P0s (dims 2, 9, 10) | 0–1 of 3 | 2–3 (dim 2 confused with Non-Goals; dims 9/10 may be flagged at wrong severity) | No reviewer reliably surfaces all 3 P0s; PM gets closest but misclassifies dim 2 |
| well-formed-standard | 0 P0s expected | 0 (correct) | — | All reviewers correctly produce only P2 polish items — no false alarms |
| internal-tool | 0 P0s; 3 N/As expected | 0 (correct) | — | N/A handling works only because PRD self-labels N/A sections explicitly; rubric-free detection is fragile |

**Overall:** Without the prd-review skill, reviewers consistently catch issues within their own domain lens (Agile Coach gets story quality right; PM gets metric vagueness right; Architecture gets NFR gaps right) but miss cross-domain P0s entirely. The three critical gaps the skill must close are: (1) no reviewer reliably catches all three P0 dimensions for `standard-with-gaps` — particularly dim 2 (Out of Scope) and dims 9/10 (GTM/Support) — because these fall between reviewer specializations; (2) N/A exemption logic is entirely absent — reviewers flag lean-PRD structural omissions as gaps rather than treating them as correct-by-design, producing noisy false-positive P1/P2 findings; (3) no reviewer produces a composite score or structured verdict label, making the output qualitative and inconsistent across fixtures. The skill's rubric, scoring formula, persona dispatch logic, and N/A exemption rules directly address all three gaps.
